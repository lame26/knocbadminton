# 🏸 KNOC 배드민턴 월례대회 관리 시스템 v3.0 (Web)

Tkinter 데스크톱 앱에서 Streamlit 웹 앱으로 전환된 버전입니다.

## 🚀 빠른 시작

### 1. 패키지 설치
```bash
pip install -r requirements.txt
```

### 2. 실행
```bash
streamlit run app.py
```

### 3. 접속
브라우저에서 `http://localhost:8501` 으로 접속

## 🔐 로그인 정보

### 슈퍼관리자
- **아이디**: `admin`
- **비밀번호**: `admin1234` (첫 로그인 후 변경 권장)

### 선수 로그인
- **아이디**: 선수 이름 (예: `배일성`)
- **비밀번호**: 사번 (예: `2143192`)

## 📱 페이지 구성

### 모든 사용자
| 페이지 | 설명 |
|--------|------|
| 🏆 실시간 랭킹 | 점수/XP 기준 순위, 통산 전적, 최근 대회 성적 |
| 📋 대진표 조회 | 날짜별 경기 일정, 조별 대진, 결과 확인 |

### 일반 선수 (Player)
| 페이지 | 설명 |
|--------|------|
| 👤 내 프로필 | 전적, 승률, 베스트 파트너, 천적, 경기 이력 |
| 🎯 내 경기 입력 | 점수 입력, 상대팀 승인/이의제기 |

### 관리자 (Admin)
| 페이지 | 설명 |
|--------|------|
| 👥 선수 관리 | 등록/수정/휴회/삭제 |
| 🏸 대진표 생성 | 참가자 선택, 매칭 방식, 자동 조편성 |
| ⚖️ 경기 중재 | 이의제기 처리, 강제 확정, 수동 경기 추가 |

### 슈퍼관리자 (Super Admin)
| 페이지 | 설명 |
|--------|------|
| 👑 권한 관리 | 관리자 지정/해제, 비밀번호 변경 |
| ⚙️ 시스템 설정 | 규칙 변경, XP 재계산, 백업/복구, 초기화 |

## 🔄 점수 입력 워크플로우

```
선수A 점수 입력 → [승인 대기] → 선수B 승인 → [확정] → 랭킹 반영
                                → 선수B 이의제기 → [관리자 중재] → 확정/삭제
```

## 📂 파일 구조

```
knoc_web/
├── app.py              # 메인 앱 (엔트리포인트)
├── config.py           # 설정 (색상, 규칙 기본값)
├── data_manager.py     # 핵심 데이터 로직
├── data.json           # 데이터 파일 (기존 호환)
├── requirements.txt    # 의존성
├── README.md           # 이 파일
└── pages/
    ├── __init__.py
    ├── page_ranking.py   # 🏆 실시간 랭킹
    ├── page_bracket.py   # 📋 대진표 조회
    ├── page_profile.py   # 👤 내 프로필
    ├── page_my_matches.py# 🎯 내 경기 입력
    ├── page_manage.py    # 👥 선수 관리
    ├── page_tourney.py   # 🏸 대진표 생성
    ├── page_mediate.py   # ⚖️ 경기 중재
    ├── page_roles.py     # 👑 권한 관리
    └── page_settings.py  # ⚙️ 시스템 설정
```

## ⚠️ 주의사항
- `data.json`은 기존 Tkinter 앱과 100% 호환됩니다.
- 웹 앱에서 추가된 필드(`role`, `input_by`, `approved_by` 등)는 기존 앱에서 무시됩니다.
- 동시 접속 시 데이터 충돌 가능성이 있으므로, 중요 작업은 관리자가 순차적으로 진행하세요.
